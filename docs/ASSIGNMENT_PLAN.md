## 2. Day별 계획

### Day 1 (완료) - 기반 구축
- json-server(`db.json`) API 연동
- 정규화/집계/지표 계산 유틸 구성
- React Query + Zustand 기본 파이프라인 연결
- 기본 대시보드 화면에서 데이터 흐름 확인

완료 기준:
- `npm run lint`, `npm run build` 통과
- `GET /campaigns`, `GET /daily_stats` 조회 확인

### Day 2 - 필수 3.1 + 3.2
- 글로벌 필터 구현
  - 기간/상태(멀티)/매체(멀티)/초기화
  - AND 조합
  - 기간 해석 기준: 집행기간과 선택기간의 **겹침(overlap)** 캠페인 포함
  - 초기값: 당월 1일~말일, 상태 전체, 매체 전체
- 일별 추이 차트 구현
  - X축 날짜, 범례, 툴팁
  - 노출/클릭 토글 + 최소 1개 선택 유지
  - 기본값: 노출+클릭 활성

완료 기준:
- 필터 변경 시 차트 데이터 즉시 동기화
- 필터 변경 시 테이블/차트 동일 결과셋 사용 확인
- 초기화 클릭 시 모든 필터가 초기값으로 복원

### Day 3 - 필수 3.3 + 3.4
- 캠페인 테이블 구현
  - 컬럼: 캠페인명/상태/매체/집행기간/총 집행금액/CTR/CPC/ROAS
  - 정렬(기간/총집행/CTR/CPC/ROAS)
  - 검색(테이블 전용), 페이지네이션(10)
  - 검색 결과 건수/전체 건수 표시
  - 체크박스 + 일괄 상태 변경
- 캠페인 등록 모달 구현
  - 유효성 검사(원문 규칙 고정)
  - `status=active`, ID 자동 생성
  - 등록 즉시 목록/대시보드 반영
  - 신규 캠페인 지표 0 또는 `-` 표기

완료 기준:
- 필수 요구사항 3.1~3.4 E2E 동작
- 집행 금액 <= 예산, 종료일 > 시작일 검증 메시지 확인
- 검색어가 차트/요약에는 영향 없음 확인

### Day 4 - 안정화 + 가산점
- 선택 과제 우선순위 구현
  1. 플랫폼 도넛(양방향 필터)
  2. Top3 랭킹
- 반응형/접근성/로딩/에러/빈 상태 보강
- 핵심 테스트 + 리팩터링
- README/AI_USAGE 최종 점검

완료 기준:
- 필수 기능 회귀 통과
- 선택 과제 최소 1개 이상 구현
- 도넛 기본 메트릭 `cost`, Top3 기본 메트릭 `ROAS` 확인
